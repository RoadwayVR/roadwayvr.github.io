Course Overview

A layer-by-layer digital twin workflow (data → model → service → user) with a final public-facing showcase.

How the course is structured

This course introduces digital twins for smart cities through a practical, project-based workflow aligned with four common layers/phases of a city digital twin:

1) Data Layer (Computer Vision): collect field traffic data from video (detection → tracking → counts/speeds).
2) Model Layer (Simulation): build a digital network and traffic model in SUMO (network → demand → signals).
3) Service Layer (Optimization): evaluate system performance using KPIs and test improvement strategies (e.g., timing changes, scenario comparison).
4) User Layer (Decision & Communication): communicate findings through clear visualizations and a public-facing project showcase (demo video + website/portfolio).

In the final weeks, students assemble selected components into a mini alternatives analysis and present results through a short demo video and a public project page.

Course Outline

What you’ll learn and how you’ll be evaluated.

Learning Goals

  • Explain what a city digital twin is and how the data/model/service/user layers connect
  • Create CV datasets (frames + labels) and run object detection for traffic scenes
  • Run object tracking and export traffic analytics (counts, speeds, trajectories)
  • Build a SUMO network and signal control, then generate a working traffic model
  • Define demand (movement/volume/speed) and compare physical vs digital behavior
  • Evaluate performance using KPIs (delay, queue, throughput, emissions) and test improvements
  • Communicate results clearly using plots, tables, and a public-facing project portfolio page

Assessment

  • Class Participation (5%)
  • In-class Deliverables (15%)
  • Transportation News Brief (10%)
  • Design Project (35%)
  • Assignments (10%)
  • Midterm Examination (paper-based) (25%)

Career Outcomes

The course provides a foundation for roles such as:

  • Digital Twin / Smart City Analyst: building and validating city-scale digital twins; connecting sensing outputs to simulation; decision support
  • Computer Vision / AI Analyst: dataset creation, detection/tracking workflows, model evaluation, and traffic analytics export
  • Traffic Simulation Analyst / Transportation Analyst: network and demand modeling, calibration reasoning, scenario testing, and KPI-based alternatives analysis
  • Applied Data / Performance Analyst: computing KPIs, comparing before/after outcomes, producing defensible summaries
  • Visualization / Planning Insights Analyst: communicating results through clear plots, tables, and portfolio-ready storytelling
